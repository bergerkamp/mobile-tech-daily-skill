# Mobile Tech Daily

每次调用生成 **一条** 移动端技术小知识，格式固定、内容精炼、可落地。

## 输出格式

严格按以下模板输出，不得增减区块：

```
📱 移动端技术小知识 · Day {N}

【{平台}】{标题}

{2-4句核心知识点，技术准确，面向中高级工程师}

💡 实践建议：{1条可直接落地的具体建议}

🔗 延伸：{1个相关关键词或技术点，供自行深入}

#{标签1} #{标签2} #{标签3}
```

其中：
- `{N}` = 本次是该会话中第几条（从1开始递增；若无上下文则填 `?`）
- `{平台}` = `Android` / `iOS` / `HarmonyOS` / `跨平台`，轮流出现，避免连续相同
- 标签从以下选取：`性能` `内存` `网络` `启动优化` `包体积` `渲染` `编译` `安全` `架构` `调试` `Kotlin` `Swift` `ArkTS` `Jetpack` `Cronet` `QUIC` `NDK` `JNI` 等

## 知识库（按主题分类）

读取 `references/topics.md` 获取完整主题库和示例知识点，从中挑选或以此为灵感生成新内容。

**选题策略：**
1. 优先选工程师日常踩坑但容易忽视的细节
2. 兼顾基础原理与最新技术动态
3. 避免过于简单（如"记得写注释"）或过于晦涩（需要500字背景才能理解）
4. 每次调用与上次平台不同（Android → iOS → HarmonyOS → 跨平台 循环）

## 批量生成

若用户要求"生成本周/N条"，则连续输出 N 条，每条之间空一行分隔，Day 编号递增。
